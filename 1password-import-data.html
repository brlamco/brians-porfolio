1.0 Importer

What is import passwords?:
Importing passwords is a new feature that enables users to seamlessly migrate credentials from their existing password manager or browser into 1Password.

Context:
Admins from within their organizations as well as new customers have limited capabilities when it comes to importing data from their existing password manager using the 1Password web app. Data migration is a core value proposition for 1Password, the web app is lacking key migration tools that would allow users to have a seamless transition and to begin moving all of their data into 1Password.

Involvement:
As the lead product designer, I was responsible for spearheading critical aspects of the password migration flow, from researching and planning to brainstorming and designing the MVP product, Alpha, Beta, and General Availability (GA) stages. Throughout the development process, I consistently worked to enhance the import functionality in the admin console application and user experience, ensuring that users can successfully migrate their credentials.

Goals:
Improve users completion rate of importing by providing teams and enterprises with a clear starting point that focused on creating the most impactful method of migration that they can use to migrate passwords. Since customers from enterprise company and small businesses have specific requirements and needs, we want to focus on helping users achieve a seamless password migration experience of moving their data from an existing service into 1Password.

Results:
As a result of making the design changes, we were able to launch the importer to the general public and saw an increase the import completion rate. We were also able to get people to migrate into 1Password faster and saw that a majority of user were able to successfully complete the entire segment of the import process upon updating each step with more educational support to fill their knowledge gaps.

Defining the problem

Before jumping into the design solution, I wanted to connect the customer problems with the business goals. I wanted to discover users current processes/decision-making about importing passwords and how they feel about the overall experience. I work closely with a UX Researcher to gather insights on the current user pain points, frustrations, and barriers related to importing into the 1Password web app.

With the support of research, we received over 40+ user feedbacks which provided understanding into what important user needs and wants the importer was not fulfilling and align requirements on what we can implement to win back churned-customers. We have categorized the problems and identified some major pain points:

Insight 01
Users were churning from the web app because we were lacking in our communication to them at crucial points in the importing process.

Insight 02
Users who were unfamiliar with the concept of importing struggled to figure out how to do basic tasks, such as exporting data from their current service or following a CSV template to prepare their file for import.

Insight 03
Users felt blindsided by certain security rules and regulations that they felt werenâ€™t expressed to them at the point of import.

6.0 Final design

Final design

On completing the usability study, we attempted more experiments and made modifications to try and improve user confidence of importing. Certain parts of the importing process was far from perfect and we want users to recognize the value of our service and address issues that is driving customer churn.

The design modifications consists of resolving issues discovered during the unmoderated usability test with users and the remaining key themes that we had wanted to focus on in our roadmap.

6.1 Import progress

Provide user feedback in response to a user action during each step of the import process.

Users were unconfident when they are deep in the importing process and are unable to determine how many steps are left until they successfully complete it. For some users, the path through the import process may be quick and simple, while for others it can be duplicative and requires a lot of mapping.

A progress indicator was implemented into the import process to help users keep track of their progress during each phase of data migration. We were able to see that users were more confident in their next steps when we use indicators because expectations were managed upfront and user have an increase in understand of how much time and effort the import process will take.

6.2 Columns awaiting to be mapped

Highlighting columns waiting to be mapped

In previous versions of the import process, users were unaware that they had to examine each column and its contentâ€”and then label them to make the import file compatible for import. Some users had assumed that the CSV file that theyâ€™ve uploaded will be mapped automatically based on their defined item type.

The importer requires the CSV file header row to determine how to map data. To help users acknowledge that these columns are requirements and need labeling for accurate data import, the corresponding column headers are highlighted in a red accent to indicate unmapped columns and to prompt users to map them if it was ignored or skipped. When columns are completely labelled, the headers are highlighted in a green accent reassuring the user that they can move forward to complete import.

Retrospective

Handling different item categories
The importer can only import passwords and items from specific password managers or browsers. With this technical limitation, it was mandatory for users moving data to 1Password to create a CSV file containing only one item category and manually perform theÂ import one at a time.

Additional guidance during data mapping
There are still untapped opportunities to improve the importer to support importing a CSV file with custom field labels and improve the speed it takes to import.

Import does not recognize duplicate items
The importer does not check whether items in the CSV file to import already exist in your vault. If the user happens to import multiple files or import a file with items that already exists in their vault,Â it will create duplicates of that item. Users would have to go into the vault where the duplicate item was imported into to remove any entries they might not need.

Item deduplication was a feature that many users had requested to see implemented into the importer. This was not easily solvable because it is difficult to determine whether if anÂ item is an accidentalÂ duplicate in the file that needs to be deleted, or if it is an intentionalÂ duplicate that needs to be preserved to import.
